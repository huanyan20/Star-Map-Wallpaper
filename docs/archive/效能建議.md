效能建議
已完成方向：星位 pool、camera cache、LOD chunk、HEALPix、分層 bloom、additive uniform registry。下一步優先看三個點：
LOD 預取與淘汰：目前 stars.js (line 123) 依 FOV、視線角度、idle 狀態載入/卸載 chunk。
可加「鄰近 chunk 預取」與「記憶體上限 LRU」，避免快速 pan/zoom 時反覆 fetch/dispose。

標籤碰撞成本：render2D.js (line 470) 已有 priority sort 和 box collision，
但 occupiedBoxes 是線性掃描。若星名開啟後標籤變多，可改成螢幕格網或 sweep-line，保留目前優先級邏輯即可。

bloom 動態降級：bloom.js (line 36) 是 5-pass，品質好但在桌布場景可能吃 fill-rate。
建議加 performance tier：低階裝置改 quarter-res blur 或降低 tap 數，高階才 full 5-pass。

測試方面我跑了 npm.cmd run test:unit：8 個測試中 6 通過、2 失敗。
失敗都在 test_additive_sky.js，原因是 Node 環境沒有 window，對應 additiveSkyMaterial.js (line 5)。
這是很值得先修的小測試相容問題。
藝術/演算法建議
你已經把前一輪很多美術方向做進去了：星光 PSF、twinkle、airmass、Purkinje、晨昏 color grading、銀河粒子、銀河 glow、星雲 decal、分層 bloom。下一輪可借鑑這些：
天象館導覽法：做「觀測主題模式」，例如月相、黃道、銀河中心、流星雨。不是新增說明文字，而是讓相關線條、標籤、光暈按優先級浮現。

攝影後製法：建立幾組 color grade preset，比如 Kaohsiung urban night、clear mountain night、twilight cinematic。不要改天文資料，只改 tone curve、bloom、skyglow、star saturation。

地圖標籤法：把星名、星座名、方位、網格標籤都納入同一套 label placement 權重，並加入「視線中心加權」和「滑鼠附近加權」。

遊戲 LOD 法：HEALPix 已在 build_tycho2.js (line 8) 使用。接下來可讓 chunk metadata 多記 magHistogram，
runtime 用 FOV 和星等分布決定載入順序，而不是只看 lod1/lod2 count。

觀測真實感：用月亮高度、月相、太陽高度、城市方向一起調整銀河/星雲可見度。
現在已有 starVisibility，可擴成 skyQuality，
統一控制 Milky Way、Nebula、dim stars、horizon haze。

我的優先順序：先修 additive sky 測試與 MCP 索引新鮮度，再做 LOD/LRU，最後做導覽模式與 color grade preset。
這樣底盤先穩，視覺提升會更可控。
