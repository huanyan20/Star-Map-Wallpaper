要將水波紋的「微觀細節」做到極致，關鍵在於打破規律性。自然界的水流絕對不是朝著單一方向等速移動的，它會受到風力、水底地形、甚至是溫度梯度的影響，產生局部的小漩渦與不規則的流速。

如果能把這套著色器（Shader）演算法調校到完美，你就能在桌面上重現如清水高美濕地那種被譽為「天空之鏡」的絕美星空倒影。在 VS Code 裡搭配你常用的輔助開發工具，這段 Shader 寫起來會非常有成就感。

以下是業界（如 AAA 遊戲引擎）實作高品質法線貼圖與 Flow Map 的核心技術與 GLSL 程式碼拆解：

一、 基礎打底：雙層法線交錯 (Dual Normal Panning)
如果你只用一張法線貼圖（Normal Map）並讓它朝一個方向移動，水面看起來會像是一塊印著波紋的輸送帶。要消除這種「塑膠感」，最便宜且高效的做法是採樣同一張法線貼圖兩次，但給予不同的縮放比例與移動速度

二、 靈魂注入：流動擾動 (Flow Mapping)
當你有了基礎的波紋後，Flow Map 是讓水面「活過來」的終極武器。

Flow Map 本身是一張 RGB 圖片，但它不代表顏色，而是代表水流的向量方向（Red 通道代表 X 軸流速，Green 通道代表 Y 軸流速）。透過它，你可以讓水波繞著礁石轉彎，或是模擬陣風吹拂水面的局部漣漪。

Flow Map 的致命陷阱：紋理拉伸
如果你直接把 Flow Map 的向量加到 UV 上並乘以時間 time，水波紋理會隨著時間無限被拉長，最後變成極度模糊的條紋。

解決方案：相位循環混合 (Phase-based Looping)
我們必須將時間拆分成兩個互相交錯的「相位（Phase）」。當 Phase 1 的紋理開始嚴重拉伸時，我們讓它淡出，同時讓剛開始循環的 Phase 2 淡入。

三、 光學細節：法線驅動的鏡面高光 (Specular Glint)
有了完美的 finalNormal 之後，微觀細節的最後一步就是光。

在深夜的星空中，如果有特別亮的星（或月亮），水面波浪的尖端會捕捉到這些光源，形成無數閃爍的微小亮點（Glints）。

## GLSL 實作策略與核心程式碼

### 1. Dual Normal Panning (雙層法線交錯)
如果沒有外部法線貼圖，我們可以利用現有的 FBM (Fractional Brownian Motion) 來生成程序化法線（Procedural Normals）。透過採樣兩次並給予不同時間縮放與方向，再將結果混合：
```glsl
// 定義兩種不同速度與縮放的 UV
vec2 uv1 = vWorldPos.xy * 2.5 + vec2(time * 0.15, time * 0.12);
vec2 uv2 = vWorldPos.xy * 4.0 - vec2(time * 0.1, time * 0.18);

// 生成程序化法線
float n1 = fbm(uv1);
float n2 = fbm(uv2);

// 混合兩層法線
vec3 bump1 = vec3(n1 - 0.5, n2 - 0.5, 0.0);
vec3 bump2 = vec3(fbm(uv2 + vec2(0.2)) - 0.5, fbm(uv1 - vec2(0.2)) - 0.5, 0.0);
vec3 combinedBump = (bump1 * 0.6 + bump2 * 0.4) * bumpStrength;
vec3 N = normalize(vNormal + combinedBump);
```

### 2. Flow Mapping (流動擾動與相位循環)
利用時間循環 (Phase-based Looping) 來解決紋理無限拉伸的問題。我們設計一個 0 到 1 循環的週期：
```glsl
float flowCycleTime = 5.0; // 循環週期 (秒)
float phase0 = fract(time / flowCycleTime);
float phase1 = fract(time / flowCycleTime + 0.5);

// 計算混合權重 (三角波：0 -> 1 -> 0)
float flowLerp = abs((0.5 - phase0) * 2.0);

// 使用 Procedural Flow Map 決定流向
vec2 flowDir = vec2(fbm(vWorldPos.xy * 0.5), fbm(vWorldPos.xy * 0.5 + vec2(10.0))) * 2.0 - 1.0;

// 將流向套用至 UV
vec2 flowUv0 = vWorldPos.xy * 2.5 + flowDir * phase0 * 0.5;
vec2 flowUv1 = vWorldPos.xy * 2.5 + flowDir * phase1 * 0.5;

// 根據 flowLerp 混合兩個相位的法線干擾
```

### 3. Specular Glint (鏡面高光)
利用法線與視角向量（View Vector），再加上高光的緊縮指數（Shininess），讓波浪尖端產生閃爍感：
```glsl
vec3 lightDir = normalize(vec3(0.5, 0.8, 0.3)); // 假定一顆明亮的主星或月亮方向
vec3 halfVector = normalize(lightDir + V);
float NdotH = max(0.0, dot(N, halfVector));

// 高光強度與範圍
float shininess = 400.0; 
float specular = pow(NdotH, shininess);

// 加上閃爍雜訊與距離衰減
float glintNoise = hash(vWorldPos.xy * 100.0 + time);
specular *= (0.5 + glintNoise * 0.5);

// 將高光疊加到水體顏色
waterColor += specular * vec3(0.8, 0.9, 1.0); // 偏藍白的星光高光
```
