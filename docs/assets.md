# STAR Assets Management

## `public/assets/` Tracking
The `public/assets/` directory is tracked in version control (without Git LFS) because the compiled assets (star catalog and label fonts) are relatively small and crucial for the project to run out of the box without requiring complex build steps for new developers.

## Build Scripts
- `build:assets`: Generates the SDF font assets and metadata required for rendering text (labels) in WebGL. It takes standard fonts as input and outputs the MSDF textures and JSON metrics to `public/assets/`.
- `build:tycho2`: Parses the raw Tycho-2 star catalog and compiles it into a binary format optimized for WebGL buffer uploads. It outputs to `public/assets/`.

## `STAR_LABEL_FONT` Premise
The `STAR_LABEL_FONT` rendering relies on the pre-built MSDF (Multi-channel Signed Distance Field) fonts generated by `build:assets`. If you change the font family or add new characters, you must re-run the build script to regenerate the MSDF textures.

## `dist/` Directory
The `dist/` directory contains the final production build generated by Vite. It is added to `.gitignore` and is not committed to the repository.
