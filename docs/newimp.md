**Stellarium Web Engine** 它的底層是 C/C++ 轉譯成 WebAssembly (WASM)，與你目前的純 WebGL/Three.js 架構不同，但它在**天文物理演算法**與**視覺渲染美學**上的原始碼，有著極度巨大的金礦可以讓你參考並移植到你的 Shader 或 JavaScript 引擎中！

### 1. 恆星色彩的科學還原 (B-V Color Index to RGB)

目前的星空引擎通常只會給星星白光，或者隨機給點藍色/黃色。Stellarium 是完全基於真實的「B-V 色澤指數」來渲染恆星顏色的。

* **參考價值**：Stellarium 內部有一套將 B-V 指數轉換為物理精確的黑體輻射色溫（Kelvin），再轉換回 sRGB 供螢幕顯示的精確查表法與公式。
* **實作應用**：你可以在打包 `stars.bin` 的階段，就用這套演算法把你的星表資料轉換好，讓 `stars.bin` 裡的 `Colors Uint8Array` 直接帶有精準的紅超巨星（如參宿四）或藍星（如天狼星）的真實色彩。

### 2. 大氣消光與折射 (Atmospheric Extinction & Refraction)

你之前提到想讓背景更真實。在真實的高雄星空中，靠近地平線的星星不僅會因為光害看不見，還會因為「大氣消光」變暗且偏紅。

* **參考價值**：Stellarium 精準實作了空氣質量（Airmass）的模型。當恆星高度角（Altitude）越低，星光穿透的大氣層越厚。
* **實作應用**：你可以參考他們的消光公式 $X \approx \frac{1}{\sin(\text{Alt})}$，將這個邏輯寫進你的 **Vertex Shader**。讓 GPU 自動把高度角小於 $15^\circ$ 的星星亮度按比例衰減並加上偏紅的色相，這會讓你的地平線視覺極度逼真。

### 3. 星光閃爍與大氣擾動 (Scintillation)

* **參考價值**：Stellarium 在模擬星星閃爍時，並不是單純的 `sin(time)`，而是考量了恆星的亮度、高度角（越靠近地平線閃得越厲害）以及觀測者設定的「大氣視寧度 (Seeing)」。
* **實作應用**：去翻閱他們處理 Scintillation 的 Shader 代碼！你可以提取出他們那套基於 Fractional Brownian Motion (fBm) 或雜訊的閃爍數學模型，直接移植到你的 WebGL Fragment Shader 裡，這比自己瞎猜參數要自然一萬倍。

### 4. 高精度星曆計算 (VSOP87)

* **參考價值**：針對太陽、月亮與八大行星的位置，簡單的三角函數在幾天內可能還算準，但拉長到幾個月就會產生嚴重偏差。Stellarium 採用了截斷版的 VSOP87 理論。
* **實作應用**：既然你已經設計了「500ms 運算快取 (astroCache)」機制，這意味著你的 CPU 有餘裕處理稍微複雜一點的數學。你可以參考他們如何簡化 VSOP87，或者直接引入現成的 JavaScript 天文庫（如 `astronomy-engine`）來提供極高精度的太陽系天體軌道。

### 5. 分區動態載入 (HEALPix 網格) - 未來擴充參考

* **參考價值**：Stellarium 使用 HEALPix（分層等面積等緯度像素化）演算法，把天球切成好幾塊。當你放大看獵戶座時，它只會載入獵戶座那區的超暗星星。
* **實作應用**：雖然你目前目標是 8785 顆肉眼可見星，全部塞進 `stars.bin` 沒問題（大約不到 200KB）。但如果哪天你想讓桌布支援放大看到 100 萬顆星，你就必須回頭參考 Stellarium 這套「按視角動態載入二進位區塊」的神級架構。
