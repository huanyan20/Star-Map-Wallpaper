# 修復黃昏幾何色塊

## Summary
- 主要問題來源鎖定在 [main.js](C:/Users/ggini/Desktop/STAR/js/main.js:789) 的黃昏光束：目前用硬邊三角形 `fill()` 疊加，太陽接近地平線時會形成截圖中的大面積幾何色塊。
- 同步修正「大氣/日光」切換狀態：目前 WebGL sky 仍持續用真實太陽位置算物理大氣，切換時容易和 2D 大氣效果不同步。

## Key Changes
- 將 `drawAtmosphericEffects` 的 crepuscular rays 從三角形改成柔化保留：
  - 不再用 `moveTo(sunP) -> lineTo(x2) -> lineTo(x3)` 的硬邊多邊形。
  - 改用低透明度、寬線條、漸層、blur/filter 的柔光束。
  - 限制只畫在天空區域，並把最大長度與 alpha 降低，避免跨過整個畫面。
- 在 [render.js](C:/Users/ggini/Desktop/STAR/js/webgl/render.js:1) 新增大氣啟用狀態傳遞：
  - `renderWebGL(..., atmosphereEnabled)`。
  - 大氣關閉時不使用太陽 daylight 強光，太陽盤與黃昏大氣效果淡出或隱藏。
- 在 [init.js](C:/Users/ggini/Desktop/STAR/js/webgl/init.js:70) 的 sky shader 增加 `atmosphereBlend`：
  - ON：使用物理大氣散射。
  - OFF：回到既有 `topRGB/midRGB/horRGB` 夜間漸層。
  - 切換時用短暫平滑過渡，避免瞬間跳色。
- 強化 sky background 的穩定性：
  - 背景材質改為不透明背景 pass，不參與透明排序。
  - 若仍有對角線 seam 風險，改成單一 full-screen triangle，避免雙三角全螢幕平面造成可見分界。

## Test Plan
- 靜態檢查：`node --check` 跑過 `main.js`、`init.js`、`render.js`、`ocean.js`。
- 本地瀏覽器驗證：
  - 用專案根目錄啟動頁面。
  - 固定時間測 `2026-07-01T18:30:00+08:00`、`18:45`、`19:15`。
  - 視角設為西方 `lookAz = 270°`，確認黃昏沒有大三角/大色塊。
- 切換驗證：
  - 連續點擊「大氣/日光」ON/OFF。
  - 確認天空、太陽盤、星等、海面光照一起切換，不出現瞬間幾何殘影。
- 視覺接受標準：
  - 黃昏可保留柔和光束，但不能有清楚直線邊界或跨畫面巨大三角面。
  - 海平線與天空漸層保持連續，星空在關閉大氣後正常可見。

## Assumptions
- 採用你選的「柔化保留」方案：保留黃昏氛圍，但完全移除硬邊幾何光束。
- 「大氣/日光」關閉時視為夜間/無 daylight 模式，而不是只關掉 2D 疊加效果。
